# Execution Observability

## Overview

The Salesforce Playwright Framework includes an execution observability layer designed to provide visibility into what happened during test execution.

Instead of relying only on Playwright stack traces and screenshots, the framework generates structured execution artifacts that capture execution behavior, business flows, test data, diagnostics, and failure context.

The goal is to make test execution easier to understand for both humans and machines.

---

# Vision

Traditional automation frameworks typically answer:

* Did the test pass?
* Did the test fail?

Execution Observability aims to answer:

* What was the test doing?
* Which business flow was executed?
* Which data was used?
* Where did the failure occur?
* Why did the failure happen?
* What should be investigated next?

---

# Generated Artifacts

The framework currently generates two main observability artifacts.

## execution-flow.json

Generated during test execution.

Purpose:

Capture execution events and diagnostics.

Location:

```text
test-results/**/execution-flow.json
```

Example:

```json
[
  {
    "type": "FLOW",
    "message": "Opening Leads page"
  },
  {
    "type": "ACTION",
    "message": "Clicking New Lead button"
  },
  {
    "type": "DATA",
    "message": "Company = IT QA"
  },
  {
    "type": "ERROR",
    "message": "Failed to click Save button"
  }
]
```

---

## playwright-results.json

Generated by Playwright.

Purpose:

Capture execution statistics and test outcomes.

Location:

```text
reports/raw/playwright-results.json
```

Provides:

* Test name
* Status
* Duration
* Project

Example:

```json
{
  "name": "should create a new lead",
  "status": "FAILED",
  "duration": 51652
}
```

---

# Reporting Pipeline

The observability layer combines multiple execution artifacts into a single report.

```text
playwright-results.json
                │
                ▼
     PlaywrightResultsParser
                │
                ▼
        Parsed Results
                │
                ▼
     ExecutionContextParser
                │
                │ enriches using
                ▼
         execution-flow.json
                │
                ▼
         ExecutionReport
                │
                ▼
      execution-report.json
```

# Execution Events

Execution Flow events are categorized into different types.

## FLOW

Represents a business flow being executed.

Example:

```json
{
  "type": "FLOW",
  "message": "Filling Lead form"
}
```

---

## ACTION

Represents an interaction performed by the automation.

Example:

```json
{
  "type": "ACTION",
  "message": "Clicking Save button"
}
```

---

## SUCCESS

Represents a successful action.

Example:

```json
{
  "type": "SUCCESS",
  "message": "Successfully clicked Save button"
}
```

---

## DATA

Represents business data used during execution.

Example:

```json
{
  "type": "DATA",
  "metadata": {
    "fieldName": "Company",
    "fieldValue": "IT QA"
  }
}
```

---

## ERROR

Represents a failure event.

Example:

```json
{
  "type": "ERROR",
  "summary": "Timeout while waiting for element",
  "executionContext": "Filling Lead form"
}
```

# Failure Intelligence

The observability layer supports failure diagnostics.

Current capabilities include:

* Root Cause Analysis
* Severity Classification
* Recommendations
* Failure Journey
* Business Impact
* Test Data Correlation

---

# Failure Journey

Failure Journey identifies:

* Business Flow
* Last Successful Step
* Failed Step

Example:

```json
{
  "failureJourney": {
    "businessFlow": "Filling Lead form",
    "lastSuccessfulStep": "Rating",
    "failedStep": "No. of Employees"
  }
}
```

---

# Test Data Visibility

Execution data is captured and correlated with failures.

Example:

```json
{
  "testData": [
    {
      "field": "Company",
      "value": "IT QA"
    },
    {
      "field": "Rating",
      "value": "Hot"
    }
  ]
}
```

Benefits:

* Easier debugging
* Faster reproduction
* Better traceability

---

# Business Impact

Business Impact provides business-level context.

Example:

```json
{
  "businessImpact": {
    "flow": "Filling Lead form",
    "failedStep": "No. of Employees",
    "recordsAffected": 1
  }
}
```
